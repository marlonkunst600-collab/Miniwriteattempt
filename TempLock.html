Mastering InnerBlocks and Template Locking

An important part of the WordPress Block Editor is its nesting capabilities. While a single block like a "Heading" or "Image" is useful, you can build more complex components using container blocks.

These are essentially blocks that hold other blocks inside, such as hero sections and testimonials. To build these, you use the InnerBlocks component.

Design Integrity: Giving users too much choice can sometimes lead to layout breaks. Template Locking is the best way to maintain consistency.

What is InnerBlocks?

InnerBlocks acts as a placeholder within your custom block where other blocks can be placed. If you've ever used the Core "Columns" or "Group" blocks, you've seen this in action.

Without guidelines, users might accidentally place a "Latest Posts" block into a "Call to Action" button wrapper. This is where Templates come in.

Defining a Blueprint with Templates

A template is a list of blocks that pre-fill your component. Here is how you would define a template for a Testimonial Card:

File: edit.js 
const MY_TEMPLATE = [
    [ 'core/image', { sizeSlug: 'thumbnail' } ],
    [ 'core/heading', { placeholder: 'Client Name', level: 3 } ],
    [ 'core/paragraph', { placeholder: 'Write the testimonial here...' } ],
];

// In your edit function:
<InnerBlocks 
    template={MY_TEMPLATE} 
/>

Template Locking

To prevent users from removing these default blocks or adding new ones, use the templateLock prop.

  • 'all': Prevents adding, removing, or reordering blocks. Users can only edit content.
  • 'insert': Prevents adding or removing blocks, but allows users to reorder existing ones.
  • 'contentOnly' (Pro Tip): Introduced in WP 6.1, this hides all design tools and only lets the user change text/images.
  • false: The default state; users have full control. 
<InnerBlocks 
    template={MY_TEMPLATE}
    templateLock="all"
/>
