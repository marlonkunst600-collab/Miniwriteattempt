Mastering FormTokenField in WordPress

When building custom blocks or plugin settings pages in WordPress, implementing multi-value inputs can be a challenge.

Whether you need a user to select multiple tags, assign team members, or list ingredients, a standard text input often leads to formatting errors, while a traditional <select multiple> is notoriously difficult to use.

What is FormTokenField?

FormTokenField is a native component found in the @wordpress/components package. It creates a "pill-style" input field where users can type a value and press enter to turn it into a token.

It supports autocomplete, keyboard navigation, and allows users to easily remove items by clicking a cross icon. If you've ever added tags to a post in the WordPress editor, you've already used this component.

Why You Should Use It

  • Native UX: Your custom block or settings page feels like a seamless part of the WordPress ecosystem.
  • Built-in Validation: It automatically prevents duplicate tokens and manages data arrays for you.
  • Accessibility: WordPress handles the complex ARIA roles and screen-reader logic required for multi-select tools.
  • Autocomplete: Provides a "suggestions" list that filters in real-time as the user types.

Basic Implementation

First, define an array attribute in your block.json:

File: block.json 
{
    "attributes": {
        "keywords": {
            "type": "array",
            "default": []
        }
    }
}
File: edit.js 
import { FormTokenField } from '@wordpress/components';

export default function Edit({ attributes, setAttributes }) {
    const { keywords } = attributes;

    const availableSuggestions = [ 'WordPress', 'JavaScript', 'React' ];

    return (
        <div className="my-custom-input-wrapper">
            <FormTokenField
                label="Select Keywords"
                value={ keywords }
                suggestions={ availableSuggestions }
                onChange={ (tokens) => setAttributes({ keywords: tokens }) }
            />
        </div>
    );
}

Dynamic Suggestions

It gets even better when you fetch suggestions dynamically. If you are building a "Featured Author" block, you can use the onInputChange prop to trigger a search against the /wp/v2/users endpoint.

Better Developer Experience

Normalize Your Data: Use the displayTransform or saveTransform props to modify text (e.g., forcing all tokens to be lowercase) to ensure database consistency.

Label Everything: Always provide a clear label prop. Visible labels are essential for accessibility and maintain a clean, standardized UI.
