Understanding Block Variations

Building custom blocks from scratch is powerful, but it is often a bit tedious to set up.

If you find yourself repeatedly configuring the same Core block settings, like setting a button to a specific brand color or choosing a specific column layout, then Block Variations can help you out.

"A Block Variation is essentially a preset for one of the existing blocks in WordPress."

To create a variation, you’ll use the registerBlockVariation function in a JavaScript file enqueued via the enqueue_block_editor_assets hook.

Creating a "Primary CTA" Variation

This code allows you to create a pre-styled "Primary Call to Action" that appears in the block inserter as if it were its own block, but it actually uses the standard WordPress Button block under the hood.

File: src/index.js 
import { registerBlockVariation } from '@wordpress/blocks';

registerBlockVariation( 'core/button', {
    name: 'primary-cta',
    title: 'Primary Call to Action',
    description: 'A high-contrast button for main site actions.',
    category: 'design',
    icon: 'admin-links',
    attributes: {
        backgroundColor: 'black',
        textColor: 'white',
        className: 'is-style-fill my-custom-cta-shadow',
        placeholder: 'Get Started Today...',
    },
