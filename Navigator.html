Understanding the __experimentalNavigator component

As your block sidebars become more complex, you will often find that users have to scroll through ten different panels just to find a single setting.

The Navigator API allows you to build an interface where users can move between different screens of settings within the same sidebar space—similar to the interface used in the WordPress Site Editor's "Styles" section.

"This allows you to navigate into 'Typography' or 'Layout' sub-menus, complete with back buttons and smooth transitions."

File: edit.js 
import { __ } from '@wordpress/i18n';
import { useBlockProps, InspectorControls } from '@wordpress/block-editor';
import {
    PanelBody,
    __experimentalNavigatorProvider as NavigatorProvider,
    __experimentalNavigatorScreen as NavigatorScreen,
    __experimentalNavigatorButton as NavigatorButton,
    __experimentalNavigatorBackButton as NavigatorBackButton,
    __experimentalVStack as VStack,
} from '@wordpress/components';

export default function Edit({ attributes, setAttributes }) {
    return (
        <InspectorControls>
            { /* 1. The Provider wraps the entire navigation system */ }
            <NavigatorProvider initialPath="/">
                
                { /* SCREEN 1: The Main Menu */ }
                <NavigatorScreen path="/">
                    <PanelBody title={ __('Settings') }>
                        <VStack spacing={4}>
                            <NavigatorButton path="/typography" variant="tertiary">
                                { __('Typography Settings') }
                            </NavigatorButton>
                        </VStack>
                    </PanelBody>
                </NavigatorScreen>

                { /* SCREEN 2: Typography Sub-menu */ }
                <NavigatorScreen path="/typography">
                    <PanelBody>
                        <NavigatorBackButton>{ __('Back') }</NavigatorBackButton>
                        { /* Settings go here */ }
                    </PanelBody>
                </NavigatorScreen>

            </NavigatorProvider>
        </InspectorControls>
    );
}

Additional Layout Helpers: VStack and HStack

These components are part of the modern WordPress Layout system. They act as a declarative way to handle alignment and spacing without writing manual CSS.

  • Consistent Grids: Instead of manual margins, <VStack spacing={4}> applies exactly 16px (4 x 4px grid) of space between all children.
  • Native RTL Support: When a user switches to an RTL language, your layout "mirrors" itself automatically.
  • The Spacer: You can drop a <Spacer /> between items to fill all available space adaptively.
Example: HStack + Spacer 
<HStack>
    <Text>Feature Name</Text>
    <Spacer />
    <ToggleControl />
</HStack>
