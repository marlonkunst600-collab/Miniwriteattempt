Mastering Dynamic Styles: CSS Variables in useBlockProps

When building custom WordPress blocks, it can be difficult to manage styles like a user-selected background color or custom spacing.

A lot of developers use inline styles, but these are hard to override and clutter the HTML. A better solution is to map block attributes to CSS Custom Properties (Variables) using the useBlockProps hook.

Why Use CSS Variables?

Instead of having to inject a hardcoded value like style="color: #ff0000", you can now inject a variable: style="--brand-color: #ff0000".

This way your code will be cleaner and you will also be able to use that variable inside complex CSS functions like calc(), rgba(), or within media queries in your style.css file.

1. The JavaScript (edit.js)

import { useBlockProps, InspectorControls } from '@wordpress/block-editor';
import { PanelBody, ColorPalette } from '@wordpress/components';

export default function Edit({ attributes, setAttributes }) {
    const { accentColor } = attributes;

    // We inject the attribute as a CSS Variable
    const blockProps = useBlockProps({
        style: {
            '--block-accent': accentColor || '#007cba',
        },
    });

    return (
        <>
            <InspectorControls>
                <PanelBody title="Settings">
                    <ColorPalette
                        value={accentColor}
                        onChange={(val) => setAttributes({ accentColor: val })}
                    />
                </PanelBody>
            </InspectorControls>
            
            <div {...blockProps}>
                <p>The border and text color are controlled by a CSS variable!</p>
            </div>
        </>
    );
}

2. The CSS (style.scss)

/* No need for inline style logic here! */
.wp-block-my-custom-block {
    border: 2px solid var(--block-accent);
    color: var(--block-accent);
    padding: 1rem;
    transition: all 0.3s ease;
}
