How to implement __experimentalLinkControl

A very common requirement when building a custom WordPress block is a link field.

Whether you are creating a "Call to Action" block, a custom button, or a linked image, you will need a way for your users to search for internal pages or paste external URLs. While you could technically use a simple text input, the user experience would not be great.

This is why I often use __experimentalLinkControl, since it provides the same, search-optimized interface used by the Paragraph and Button blocks from the WordPress Core.

What is LinkControl?

LinkControl is a high-level component within the @wordpress/block-editor package. It includes search suggestions, support for external URLs, and a UI that looks like the rest of WordPress. You can also add toggles for "Open in New Tab" or "No-follow" attributes.

No longer Experimental

For the last few years we had to access this component via the __experimentalLinkControl export. As of WordPress 6.8 (Gutenberg 19.9), the component has officially stabilized. If you are working on a modern project, you can now import it directly as LinkControl.

Implementing LinkControl

First, define an attribute in your block.json to store the link data (usually as an object):

File: block.json 
{
    "attributes": {
        "link": {
            "type": "object",
            "default": { 
                "url": "", 
                "opensInNewTab": false 
            }
        }
    }
}
File: edit.js 
import { __experimentalLinkControl as LinkControl } from '@wordpress/block-editor';

export default function Edit({ attributes, setAttributes }) {
    const { link } = attributes;

    return (
        <div { ...useBlockProps() }>
            <LinkControl
                value={ link }
                onChange={ (newLink) => setAttributes({ link: newLink }) }
                settings={[
                    {
                        id: 'opensInNewTab',
                        title: 'Open in new tab',
                    },
                ]}
                suggestionsQuery={{
                    type: 'post',
                    subtype: 'page',
                }}
            />
        </div>
    );
}

Why Use suggestionsQuery?

One of the best features of LinkControl is the suggestionsQuery prop. By default, the search looks through all public post types.

By passing { type: 'post', subtype: 'product' }, you can create a better experience that prevents users from accidentally linking to the wrong content type.
